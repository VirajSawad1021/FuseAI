% ── Database Setup ────────────────────────────
\section{Database Setup}

The same PostgreSQL database and dataset from previous sessions are used throughout all tasks. The database is the \textbf{ClassicModels} schema, which contains 8 tables:

\begin{itemize}[leftmargin=1.5cm]
    \item \texttt{customers}, \texttt{orders}, \texttt{orderdetails}
    \item \texttt{products}, \texttt{productlines}
    \item \texttt{employees}, \texttt{offices}
    \item \texttt{payments}
\end{itemize}

The database is seeded via \texttt{seed.sql} and runs on \textbf{PostgreSQL 16} inside a Docker container.

% ── Implementation ────────────────────────────
\section{Implementation}

The system is built using \textbf{FastAPI} (Python) and exposes a REST API. The full source code is organized inside the \texttt{Assignment\_3/} folder of the repository.

\subsection*{Task 1 — SQL Benchmark \& Evaluation Design}
Ground truth SQL queries were written for all benchmark questions from the provided dataset. An evaluation framework was designed to assess the agent across dimensions including execution success rate, SQL correctness, join accuracy, and self-correction performance.

The evaluation strategy document is available at:\\
\href{https://github.com/VirajSawad1021/FuseAI/blob/main/Assignment_3/task1_evaluation_strategy.md}{\texttt{task1\_evaluation\_strategy.md}}

\subsection*{Task 2 — Query Understanding (Decomposition)}
Each natural language question is decomposed into structured components before SQL generation:
\begin{itemize}
    \item \textbf{Intent} — what is being asked
    \item \textbf{Tables} — which tables are needed
    \item \textbf{Columns} — which columns to retrieve
    \item \textbf{Filters} — WHERE conditions
    \item \textbf{Joins} — how tables are connected
\end{itemize}
This decomposition is performed by \textbf{GPT-4o-mini} via prompt chaining (\texttt{app/sql\_generator.py}).

\subsection*{Task 3 — Text-to-SQL Pipeline}
The pipeline follows these steps:

\begin{enumerate}
    \item Accept a natural language question via \texttt{POST /agent/sql}
    \item Decompose the question using an LLM prompt
    \item Generate a PostgreSQL \texttt{SELECT} query from the decomposition
    \item Validate the query (block destructive operations)
    \item Execute the query against PostgreSQL
    \item Return structured results
\end{enumerate}

\subsection*{Task 4 — Mini SQL Agent (Agentic System)}
The agent adds self-correction and retry logic on top of Task 3:

\begin{itemize}
    \item On query failure, the error message is fed back to the LLM
    \item The agent attempts to fix and retry the query (up to \textbf{3 retries})
    \item On final success, results are summarised in natural language
    \item All steps are logged with timestamps and execution time
\end{itemize}

\textbf{Safety rule:} Only \texttt{SELECT} and \texttt{WITH} queries are permitted. Any \texttt{DROP}, \texttt{DELETE}, \texttt{UPDATE}, \texttt{INSERT}, or \texttt{ALTER} is blocked by the validator.
